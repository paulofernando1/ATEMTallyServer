# Troubleshooting

This page lists common technical issues encountered when setting up Tally Server Pro and how to resolve them quickly.

## 🛠️ General Issues

### 1. Server Does Not Start
-   **Error:** "Address in use (9910)".
-   **Cause:** Another application is using UDP port 9910. Common culprits are Blackmagic Desktop Video, Tally Arbiter, or other tally software.
-   **Solution:** Close conflicting software or change the Tally Server Pro port in the Settings (though hardware units will need re-flashing).

### 2. Dashboard Is Blank
-   **Cause:** The application hasn't loaded a configuration yet.
-   **Solution:** Go to **File** -> **Open Layout** or go to **Settings** -> **Active Network Clients**, and add at least one tally row. Or, simply start the server and wait for a tally unit to connect.

---

## 📡 Networking Issues

### 1. Tally Units "Seeking Switcher..."
-   **Cause:** The ESP8266/ESP32 doesn't have the correct Server IP.
-   **Solution:** Verify the `switcherIp` in your `.ino` (or `.cpp`) code and flash the device again.
-   **Firewall:** Ensure Windows Firewall allows the Tally Server Pro `.exe` to receive UDP packets on port 9910.

### 2. Remote SID Shows "Client 1234..."
-   **Cause:** This is normal! SIDs are long hashes generated by Socket.IO. We've enhanced the display in v1.2 to show the first 8 characters for better identification.

---

## 📱 Web Tally Issues

### 1. "FAILED: Library not loaded"
-   **Cause:** The mobile device is attempting to load a cached or broken version of the Socket.io library.
-   **Solution:** Refresh the mobile browser or click **Force Re-Sync** on the mobile screen. Our latest versions (v1.2+) address this with a cache-busting version system.

### 2. Connecting... Ad Infinitum
-   **Cause:** WiFi Isolation. Some production routers prevent wireless clients from talking to each other (AP Isolation).
-   **Solution:** Ensure AP Isolation is **Disabled** on your WiFi router.

---

## 🔗 Integrated Switchers (OBS / vMix)

### 1. vMix Connection Failed
-   **Cause:** vMix is not running or vMix Tally Port (default `8099`) is disabled.
-   **Solution:** Ensure vMix is open and that internal Tally reporting is active.

### 2. OBS WebSocket Error
-   **Cause:** Password mismatch or OBS Websocket version is too old.
-   **Solution:** Tally Server Pro requires **OBS WebSocket 5.x** (Standard in OBS 28+). Check your OBS WebSocket settings, copy the password, and paste it into the Emulator settings.
